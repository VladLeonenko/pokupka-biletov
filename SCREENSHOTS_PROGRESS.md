# Прогресс создания скриншотов для кейсов

## Запуск скрипта

Скрипт автоматически создает скриншоты для всех кейсов:

```bash
cd backend
node scripts/createScreenshots.js
```

Для тестирования на 5 кейсах:
```bash
node scripts/createScreenshots.js --test
```

## Что делает скрипт

1. **Обложка (cover.png)**: Скриншот главной страницы сайта донора
2. **Галерея 1 (gallery-1.png)**: Скриншот главной страницы (дубликат для галереи)
3. **Галерея 2 (gallery-2.png)**: Скриншот каталога/услуг или средней части главной страницы
4. **Галерея 3 (gallery-3.png)**: Скриншот контактов или нижней части главной страницы

## Структура файлов

Все изображения сохраняются в:
```
frontend/public/legacy/img/cases/{slug}/
  ├── cover.png
  ├── gallery-1.png
  ├── gallery-2.png
  └── gallery-3.png
```

## Особенности

- Скрипт пропускает уже существующие изображения
- Автоматически пытается найти каталог и контакты на сайте
- Если не находит, делает скриншоты разных частей главной страницы
- Обрабатывает ошибки и продолжает работу даже если некоторые сайты недоступны
- Делает паузу 2 секунды между кейсами, чтобы не перегружать серверы

## Время выполнения

Для 258 кейсов потребуется примерно:
- 258 кейсов × 4 изображения × ~5 секунд = ~1.5 часа

## Мониторинг прогресса

Проверить прогресс можно через:
```bash
tail -f /tmp/createScreenshots.log
```

Или проверить количество созданных изображений:
```bash
find frontend/public/legacy/img/cases -name "*.png" | wc -l
```

## После завершения

После завершения скрипта все кейсы будут иметь:
- ✅ Обложки (cover.png)
- ✅ Галерею из 3 скриншотов (gallery-1.png, gallery-2.png, gallery-3.png)
