### 1. 🍪 БАННЕР СОГЛАСИЯ НА COOKIES (Приоритет: ВЫСОКИЙ)

**Описание:**
Модальное окно/баннер, появляющийся при первом посещении сайта с уведомлением о сборе cookies.

**Требования:**
- Появляется только при первом посещении (проверка в localStorage)
- Не блокирует контент (можно закрыть или проигнорировать)
- Кнопка "Принять все"
- Кнопка "Настроить" (открывает расширенные настройки)
- Кнопка "Отклонить" (только обязательные cookies)
- Ссылка на политику конфиденциальности

**Расположение:**
- Внизу экрана (sticky banner) или центрированное модальное окно
- Адаптивный дизайн для мобильных

**Компонент:** `CookieConsentBanner.tsx`

### 2. ⚙️ РАСШИРЕННЫЕ НАСТРОЙКИ COOKIES (Приоритет: ВЫСОКИЙ)

**Описание:**
Модальное окно с детальными настройками типов cookies.

**Типы cookies для настройки:**
1. **Обязательные (нельзя отключить)**
   - Авторизация
   - Корзина
   - Безопасность
   - Настройки сайта

2. **Функциональные (можно отключить)**
   - Сохранение предпочтений
   - Языковые настройки
   - История просмотров

3. **Аналитические (можно отключить)**
   - Google Analytics
   - Яндекс.Метрика
   - Статистика посещений

4. **Маркетинговые (можно отключить)**
   - Персонализированная реклама
   - Ретаргетинг
   - Отслеживание конверсий

**Функционал:**
- Чекбоксы для каждого типа (кроме обязательных)
- Описание каждого типа
- Кнопка "Сохранить настройки"
- Кнопка "Принять все"
- Кнопка "Отклонить все" (кроме обязательных)

**Компонент:** `CookieSettingsModal.tsx`

### 5. 🔐 ХРАНЕНИЕ СОГЛАСИЙ (Приоритет: ВЫСОКИЙ)

**Backend API:**
- `POST /api/consents` - сохранение согласий пользователя
- `GET /api/consents` - получение согласий пользователя
- `PUT /api/consents/:id` - обновление согласия
- `DELETE /api/consents/:id` - отзыв согласия

**Frontend:**
- Хранение в localStorage (для неавторизованных)
- Хранение в БД (для авторизованных)
- Синхронизация при авторизации

**Структура данных:**
```typescript
interface Consent {
  id: number;
  user_id?: number;
  session_id?: string; // для неавторизованных
  type: 'cookies' | 'privacy' | 'marketing' | 'analytics';
  category: 'necessary' | 'functional' | 'analytical' | 'marketing';
  accepted: boolean;
  ip_address?: string;
  user_agent?: string;
  created_at: string;
  updated_at: string;
}
```
