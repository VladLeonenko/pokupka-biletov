## 1. 🔴 SEO Position Checker (Критично)

### Проблема
```javascript
// backend/routes/seoTools.js, строки 264-272
if (keywordInTitle) {
  position = Math.floor(Math.random() * 10) + 1; // ❌ РАНДОМ!
} else if (keywordInH1) {
  position = Math.floor(Math.random() * 15) + 5; // ❌ РАНДОМ!
} else if (keywordInMeta) {
  position = Math.floor(Math.random() * 20) + 10; // ❌ РАНДОМ!
} else {
  position = Math.floor(Math.random() * 30) + 20; // ❌ РАНДОМ!
}
```

### Что делает ПРАВИЛЬНО ✅
- Загружает реальный HTML сайта
- Проверяет наличие keywords в title, H1, meta description, content
- Валидирует URL

### Что делает НЕПРАВИЛЬНО ❌
- Генерирует **случайные** позиции через `Math.random()`
- Не использует реальные API поисковых систем
- Не сохраняет историю проверок
- Каждый запуск даёт разные результаты

### Решение 🎯
**Алгоритм расчёта реалистичной позиции:**

1. **Частота упоминания ключевого слова** (вес: 30%)
   - Считаем сколько раз слово встречается в HTML
   - Чем больше - тем выше позиция

2. **Релевантность размещения** (вес: 40%)
   - Title = 100 баллов
   - H1 = 80 баллов
   - Meta description = 60 баллов
   - H2-H6 = 40 баллов
   - Body text = 20 баллов

3. **SEO качество страницы** (вес: 20%)
   - Длина title (оптимально 30-60 символов)
   - Длина description (оптимально 120-160 символов)
   - Наличие alt у изображений
   - Скорость загрузки

4. **Плотность ключевых слов** (вес: 10%)
   - Оптимально: 1-3% от общего текста
   - Переоптимизация снижает позицию

**Формула:**
```
score = (frequency * 0.3) + (relevance * 0.4) + (seo_quality * 0.2) + (density * 0.1)
position = Math.max(1, Math.min(100, 100 - score))
```

```javascript
// backend/routes/seoTools.js, строки 470-496
const simulatedMentions = Array.from({ length: Math.floor(Math.random() * 3) + 1 }, (_, i) => {
  // ❌ Генерация ФЕЙКОВЫХ упоминаний

    source: sources[Math.floor(Math.random() * sources.length)],
```
