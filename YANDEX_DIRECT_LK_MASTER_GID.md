# Мастер-гид: Яндекс Директ под конверсии с первого дня

Документ описывает **полную иерархию** аккаунта в том виде, в каком вы её строите: **5 кампаний**, в первой — **4 группы по 3 объявления**; в остальных четырёх — **по 3–4 группы**, в каждой **по 3–4 объявления**. Ниже — **каждый экран и поле**, лимиты **заголовок 55 / доп. заголовок 30 / текст 81** символ, и **готовые креативы** там, где их можно стандартизировать.

Главная цель с точки зрения рекламы: не «клики», а **заказы** — в Метрике это цели **`purchase`** и опорная **`checkout_start`**, пока мало покупок.

**Суперфинал (Спартак — Краснодар, 24.05.2026):** отдельный контур — **2 поисковые кампании**, **30–40k ₽**, **8 объявлений** (2 группы × 2 объявления). Пошагово в ЛК: **раздел 11**. Бюджет и минусы: **`BOEVOY_MEDIAPLAN_YANDEX_DIRECT.md` → раздел 18**.

## 3. Уровень кампании: поля и правила «день 1»

Заполняйте **сверху вниз** в черновике кампании.

### 3.1 Идентификация и ссылка по умолчанию

- **Название кампании:** короткое, латиница/транслит, без пробелов в начале — удобно для UTM `{campaign_name}` и отчётов.
- **Главная ссылка / «что продвигаете»:** для событийных кампаний — **карточка конкретного спектакля**, не главная сайта.

Пример Кабала:

`https://biletvsem.com/ticket/686cd69c58f79d0030278b9d/kabala-svyatosh`

Для Гамлета и общей афиши подставьте **фактические** URL карточек или раздела (в этом файле оставлены плейсхолдеры там, где ID у вас другой).

**Суперфинал (все объявления SF):**

`https://biletvsem.com/ticket/superfinal-fonbet-kubka-rossii-spartak-krasnodar`

Резерв: `https://biletvsem.com/ticket/6a05d17b46a4d000309ecf4e/superfinal-fonbet-kubka-rossii-spartak-krasnodar`  
repertoireId: `6a05d17b46a4d000309ecf4e`

### 3.2 Места показа

- **Поисковые кампании:** Поиск Яндекса + при необходимости динамические места на поиске; **РСЯ и Карты выкл**.
- **РСЯ-кампании:** наоборот — только сеть, поиск выкл.

Смешивание Поиска и РСЯ в одной кампании на старте **не рекомендуется** — разная экономика и разбор запросов.

### 3.3 Стратегия и бюджет

- Старт: **максимум конверсий** (или эквивалент по конверсиям) при подключённой **Метрике** и цели **`purchase`**.
- **Один** счётчик **`109119282`** в целях; цели с **других** счётчиков не подмешивать.
- **Ориентир CPA** по `purchase` и **макс. ставка** задайте заранее (чтобы не разгоняли клики без покупок).
- **Расписание:** без ночи, например **08:00–23:30**; выходные и переносы — как в календаре Директа (логика «выходной как будний» уже в интерфейсе).

### 3.4 UTM на кампании (база)

Шаблон (копипаст):

```text
utm_source=yandex&utm_medium=cpc&utm_campaign={campaign_id}_{campaign_name}&utm_content={ad_id}_{phrase_id}&utm_term={keyword}
```

На **группу** и **объявление** добавляйте только **дополнительные** метки (`utm_content` для группы, `utm_term=adN` для объявления), **не дублируя** `source`/`medium`.

### 3.5 Минус-слова (кампания)

Общий список для всех поисковых театральных кампаний (копипаст):

```text
бесплатно скачать торрент epub fb2 txt краткое содержание реферат сочинение доклад школьный википедия фильм сериал читать онлайн pdf youtube кинотеатр тв программа аудиокнига
```

Дополняйте из отчёта **«Поисковые запросы»** ежедневно первую неделю.

### 3.6 Корректировки (кампания)

- Мобильный трафик: **+15–20%** (типично для билетов).
- Возраст: после первых данных; на старте можно слегка усилить **25–54**, ослабить **до 18**, если нет детских постановок в этой РК.

### 3.7 Доп. элементы на кампании

- **Быстрые ссылки** и **уточнения** можно задать один раз на кампанию и **наследовать** в группах/объявлениях.
- **Промоакция** — только при реальной акции.
- **Автоприменение рекомендаций Директа** — лучше **выкл** на период обкатки.

### 3.8 Мониторинг сайта

**Вкл** — при падении сайта показы останавливаются.

## 4. Уровень группы: поля и логика таргетинга

### 4.1 Название группы

Читаемое имя для отчётов: `kabala_hot`, `kabala_afisha`, `kabala_brand`, `kabala_geo` и т.п.

### 4.2 Схема «Взаимодействие таргетингов на Поиске»

Имеет вид:

**География ∧ (Автотаргетинг ∨ Ключевые фразы) ∧ Сегменты аудитории.**

- **Сегменты аудитории пустые** — в группах «холодного/горячего поиска» по умолчанию; иначе **И** с ключами резко режет охват.
- Ретаргетинг — **отдельная группа или кампания**.

### 4.3 География

Москва; при необходимости **Москва + МО**. Не вся Россия.

### 4.4 Сценарий группы

На старте: **«Вся заинтересованная аудитория»** — больше данных для стратегии.

### 4.5 Автотаргетинг

Для узких групп (горячий интент): **целевые + узкие** вкл, **широкие / сопутствующие / альтернативные** выкл до появления статистики.  
Конкуренты по брендам — **выкл** на старте.

### 4.6 Ключевые и минус-фразы

Одна группа = **одно намерение**. Ключи не дублировать между группами одной кампании (конкурируют сами с собой).

### 4.7 Быстрые ссылки и UTM группы

- Быстрые ссылки: **карточка события**, афиша, возврат, контакты — URL проверить вручную по клику.
- UTM группы: например `utm_content=kabala_hot` (без повторения базовых UTM).

### 4.8 Корректировки на группе

Только если отличаются от кампании; иначе не дублировать.

## 5. Уровень объявления: каждое поле формы

### 5.1 Тип объявления

**Текстово-графическое** — основной тип для Поиска под ваши креативы.  
**Комбинаторное** — позже, когда накопите выигрышные формулировки.  
**Нейрообъявление** — не как единственный источник текстов: проверять факты и цены.

### 5.2 Ссылка в объявлении

Всегда **та же карточка события**, что и смысл группы (для «Кабалы» — URL Кабалы).

### 5.3 Параметры URL

Если UTM на кампании и группе уже есть — в объявлении, например:

```text
utm_term=ad1_g1
```

Меняйте для каждого объявления (`ad2_g1`, …).

### 5.4 Генерация объявления

Можно использовать для **черновика**, итог **только ручная вычитка**. Не оставлять выдуманные цены и даты.

### 5.5 Заголовок (55) · Доп. заголовок (30) · Текст (81)

См. раздел **7** — готовые наборы по группам.

### 5.6 Изображение

Квадрат **от 450×450** пикселей; афиша без несуществующей цены крупным шрифтом.

### 5.7 Видео

Для чистого Поиска необязательно; для смешанных форматов — **5–15 с**, MP4.

### 5.8 Карусель

Имеет смысл в **РСЯ**; для поиска — по желанию.

### 5.9 Кнопка

Для РСЯ: «Выбрать места» / «Купить билет», ссылка на карточку.

### 5.10 Контакты (Яндекс Бизнес)

Подключать, если организация в Бизнесе **совпадает** с тем, как вы рекламируете продажу билетов.

### 5.11 Цена

Только если на посадочной **то же значение** («от N ₽» — только если везде честно).

### 5.12 Уточнения (до 25 символов на строку)

Примеры строк (подгонка под лимит в интерфейсе):

- Электронный билет  
- Выбор мест в зале  
- Оплата онлайн  
- Даты на странице события  
- Возврат — правила на сайте  
- Поддержка в контактах  

Либо «Использовать из группы/кампании», если там задано.

### 5.13 Быстрые ссылки

«Использовать из группы/кампании» — ок при верных URL.

### 5.14 ЕРИР / описание для маркировки

Включайте автогенерацию, если так требует интерфейс; после смены текста объявления **перепроверяйте** блок.

### 5.15 Индикатор качества

Доводите до заполненных **уточнений** и рабочих **быстрых ссылок**; финальная оценка — по **конверсии и CPA**, а не только по «зелёности» мастера.
