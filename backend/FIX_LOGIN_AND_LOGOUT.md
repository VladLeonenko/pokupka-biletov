# Исправление проблем логина и выброса из ЛК

## Проблема 1: Постоянное обновление страницы при логине

**Причина:** Конфликт между несколькими редиректами:
- `useEffect` в LoginPage редиректит при наличии token && user
- `onSubmit` тоже делает navigate после логина
- Это создавало цикл редиректов

**Исправление:**
- ✅ Добавлен флаг `isRedirecting` для предотвращения множественных редиректов
- ✅ `useEffect` срабатывает только если не идет процесс логина
- ✅ `onSubmit` делает редирект один раз после успешного логина
- ✅ Убрано дублирование логики редиректа

## Проблема 2: Выбрасывает из ЛК через какое-то время

**Возможные причины:**
1. **JWT токен истекает** - токен живет 7 дней, но если токен истек, нужно автоматически разлогинить
2. **401 ошибки не обрабатываются** - когда токен истек, API возвращает 401, но фронтенд не обрабатывает это глобально
3. **useCacheVersionWatcher** - может очищать localStorage, но он сохраняет 'auth.token', так что это не должно быть проблемой
4. **Периодическая проверка токена** - нужно проверять валидность токена периодически

**Исправления:**
- ✅ Добавлена глобальная обработка 401 в `doFetch` (cmsApi.ts и ecommerceApi.ts)
- ✅ При 401 автоматически очищается auth и редирект на логин
- ✅ Добавлена периодическая проверка валидности токена в AuthProvider (каждые 5 минут)
- ✅ При истечении токена автоматический logout и редирект

**Аналитика выброса из ЛК:**

### Проверка на сервере:

```bash
# Проверить логи бэкенда на 401 ошибки
pm2 logs primecoder-backend --lines 200 | grep -i "401\|unauthorized\|invalid token"

# Проверить время жизни токена в middleware/auth.js
# Токен живет 7 дней (expiresIn: '7d')
```

### Проверка в браузере:

1. Откройте DevTools → Console
2. При выбросе из ЛК проверьте:
   - Есть ли ошибки `401 Unauthorized`
   - Есть ли логи `[AuthProvider] Token expired or invalid`
   - Есть ли логи `[cmsApi] 401 Unauthorized - clearing auth`

### Возможные причины выброса:

1. **Токен истек (7 дней)** - нормальное поведение, нужно перелогиниться
2. **JWT_SECRET изменился** - токены стали невалидными
3. **Проблемы с сетью** - временные 401 из-за проблем с подключением
4. **useCacheVersionWatcher** - если версия кэша изменилась, происходит reload, но токен сохраняется

### Решение для продления сессии:

Если нужно продлить время жизни токена, измените в `backend/middleware/auth.js`:
```javascript
return jwt.sign(payload, JWT_SECRET, { expiresIn: '30d' }); // Вместо '7d'
```

Но это не рекомендуется с точки зрения безопасности.

### Лучшее решение - refresh token:

В будущем можно добавить refresh token механизм:
1. Access token живет 1 час
2. Refresh token живет 30 дней
3. При истечении access token автоматически обновляется через refresh token
4. Только при истечении refresh token требуется перелогин

## Проверка после исправлений

1. **Логин:**
   - Войти в админ-панель
   - Не должно быть циклов обновления страницы
   - Редирект должен быть один раз

2. **Выброс из ЛК:**
   - Проверить логи в консоли браузера
   - Проверить логи на сервере
   - Если выброс происходит, проверить причину (401, истечение токена, и т.д.)
