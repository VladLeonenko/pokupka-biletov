## Исходная задача

В SVG у каждого `circle` нужны корректные **ряд** (`row` / `data-row`) и **место** (`place` / `data-seat` / `data-seat`). Геометрия Лужников нерегулярная: террасы, лесенки, проходы, дуги, зеркальные сектора, на одном X часто **два круга** (`fieldGrid` + `strict`).

Ручной инструмент с линией вдоль ряда уже был; цель диалога — **массовое заполнение сектора** без перезаписи хороших рядов и с валидацией.

**Вывод на старте:** автоматом на весь стадион без косяков нельзя; реалистично — сектор за сектором, визуальная проверка, красные «без ряда», сохранение на сервер.

## Что добавили в инструмент (фичи)

### Первая волна (`e335850b` — sector labeling helpers)

- **↕ сектор** — предложение разметки всего сектора, preview (бирюза), confirm перед записью.
- **✓ проверка** — подсветка: без ряда/места, дубли `сектор+ряд+место`, пропуски в нумерации, скачки порядка мест, точки далеко от оси ряда.
- **⇄ копия** — копирование разметки в другой сектор с реверсом мест.
- **Confirm пачками** для обычного применения ряда по линии («Похоже, A116, ряд 6, места 1–34»).
- **Zoom/pan** — зум к курсору, меньше «чёрного экрана».
- `circlesForSector`, `groupedLabeledRows`, нормализация секторов.

### Вторая волна (точность ↕)

- Подписи **рядов слева** (`renderRowLabels`) — только в редакторе, не в сохранённом SVG.
- Поля **Ряд** / **по** (диапазон рядов); для колонок — не путать с «место с … по …».
- Типы инструмента: **1-я колонка**, **посл. колонка**, **ряд** (▶ на крайних рядах).
- **⌫ ряд/место** — сброс row/place и `data-sector-edge` (`d07bde28`).
- **▢ область** — рамка, очистка внутри (`482bbc3e`).
- Защита **якорей колонок** (`data-sector-edge`) — ↕ не перезаписывает контур.
- Заполнение **interior** даже если на точке был старый `manual-row` (`c97e6547`).

## Канон workflow ↕ сектор (как договорились в диалоге)

1. Поле **Сектор** — точный код (`C136 VIP`, `C142`, `Сектор C 142` → нормализация `c142`).
2. **Тип «1-я колонка»** — клики по **левой** (min X) колонке на **≥2 рядах** (лучше низ, верх, ступенька).
3. **Тип «посл. колонка»** — клики по **правой** (max X) колонке на тех же рядах. **Не писать** `place` на ▶ посл. колонки — номер посчитает ↕.
4. Опционально **Ряд** / **по** — если пусто или только один ряд, диапазон берётся из якорей колонок (`resolveSectorFillRowSpan`).
5. Сложные ряды можно разметить вручную **до** ↕; якоря колонок после **⌫** нужно проставить заново.
6. **↕ сектор** → бирюзовый preview → **Применить ↕** → сохранение на сервер.
7. **✓ проверка** — «пропуски N» = дыры в нумерации мест внутри ряда, не «0 предложений».

**«Место с»** при типе колонка: для apply колонки по Shift+клику; для ↕ сектор **не задаёт** сетку мест — только якоря X и номер ряда на точке.

**Направление:** ряды по **Y** (полосы), места в ряду по **X** (слева направо относительно центра поля / от 1-й колонки к посл.).

| 0 предложений | Все точки считались manual/контур; VIP `c136` ≠ `vipc136` | Interior fill; `sectorNormsMatch` + bbox |

## Алгоритм ↕ сектор (актуальный, после `083f240e`)

1. `circlesForSector(label)` — `data-sector` / `place-name` + bbox; VIP alias.
2. `collectColumnAnchors(..., 'first'|'last')` — только `data-sector-edge`; в ряду min X / max X.
3. `resolveSectorFillRowSpan` — min/max ряд из якорей ± поля «Ряд»/«по».
4. `clusterBandsByY` → для **каждой** точки `rowNumForCircle` (Y + `rowNumFromBandNear` по нормали к оси якорей 1-й колонки).
5. В ряду: `filterRowDotsBetweenAnchors` (проход ~60px при pitch ~20 на C142); `snapRowAnchorsToDots` для интерполяции.
6. `dedupeSeatDotsByX` — одна колонка = один слот; rank: edge > strict > fieldGrid.
7. Нумерация: место 1 на 1-й колонке (`allowFirstTwin` на strict-двойник); interior `seatNum++`; посл. — `resolveLastSlot` (▶ last или крайний X).

### Ключевые функции

- `rowAxisFromColumnAnchors`, `anchorNormalProj`, `rowNumFromBandNear`
- `filterRowDotsBetweenAnchors`, `snapRowAnchorsToDots`, `resolveLastSlot`
- `circlesForSeatAssign`, `appendSeatToSlot`
- `inferSectorLabelForApply` — подпись сектора как в SVG (VIP)

## Сектор C142

**Геометрия (из `luzhniki-grid-diagnostic.html`):**

- Ряды **1–15 / 16–26 / 27–39** — разная ширина (лесенка справа).
- Шаг ряда по Y ≈ **21.7px**.
- Ряд **16:** два блока **4 + 10** мест, проход **~60px** (pitch ~20).
- Ряд **39:** до **50** мест в одной полосе по X.

**Типичные ошибки пользователя (по скринам):**

- Якоря в **центре** сетки → ↕ заполняет вертикальную полосу.
- Якоря только на одном блоке → правый блок после прохода без номеров.

**Что делать:**

1. `C142` в поле сектора.
2. 1-я / посл. колонка на **реальных** краях: низ (1), верх (39), ряд **16** или **27** на ступеньке.
3. ⌫ старые row/place в секторе.
4. ↕ → проверка → применить.

**Коммиты:** `82d7b8e7` (Y-полосы), `083f240e` (точка + aisle + worklog).
