# Полная аналитика проблем авторизации

## Проблема 1: Постоянное обновление страницы при логине ✅ ИСПРАВЛЕНО

**Симптомы:**
- При попытке логина страница постоянно обновляется
- Бесконечный цикл редиректов

**Причина:**
Конфликт между несколькими редиректами:
1. `useEffect` в LoginPage редиректит если `token && user && !loading && !email && !password`
2. `onSubmit` делает `navigate` после успешного логина
3. `Protected` компонент может редиректить обратно на `/admin/login` если `user` еще не загружен

**Исправление:**
- ✅ Добавлен флаг `isRedirecting` для предотвращения множественных редиректов
- ✅ `useEffect` срабатывает только если не идет процесс логина (`!loading && !isRedirecting`)
- ✅ `onSubmit` делает редирект один раз после успешного логина
- ✅ `Protected` компонент ждет загрузки `user` (500ms timeout) вместо немедленного редиректа

## Проблема 2: Выбрасывает из ЛК через какое-то время ✅ ИСПРАВЛЕНО

**Симптомы:**
- Пользователь работает в ЛК
- Через некоторое время (несколько минут/часов) его выбрасывает
- Нужно снова логиниться

**Возможные причины:**

### 1. JWT токен истекает (7 дней)
- **Токен живет:** 7 дней (`expiresIn: '7d'` в `backend/middleware/auth.js`)
- **Проверка:** Если прошло больше 7 дней с последнего логина - нормальное поведение
- **Решение:** Увеличить время жизни токена (не рекомендуется) или добавить refresh token

### 2. 401 ошибки не обрабатываются глобально
- **Проблема:** Когда токен истек, API возвращает 401, но фронтенд не обрабатывает это глобально
- **Решение:** ✅ Добавлена глобальная обработка 401 в `doFetch` (cmsApi.ts и ecommerceApi.ts)
  - При 401 автоматически очищается `auth.token` и `auth.user`
  - Редирект на `/admin/login` (если не на странице логина)

### 3. useCacheVersionWatcher очищает localStorage
- **Проверка:** `useCacheVersionWatcher` сохраняет `auth.token` и `token` в `PRESERVE_KEYS`
- **Вывод:** Это НЕ причина выброса, токен сохраняется при обновлении кэша

### 4. Периодическая проверка валидности токена
- **Проблема:** Нет автоматической проверки валидности токена
- **Решение:** ✅ Добавлена периодическая проверка в `AuthProvider` (каждые 5 минут)
  - Проверяет валидность токена через `getCurrentUser()`
  - При 401 автоматически logout и редирект

### 5. Проблемы с сетью/сервером
- **Проверка:** Временные 401 из-за проблем с подключением
- **Решение:** ✅ Обработка 401 добавлена, но нужно проверить логи на сервере

## Диагностика выброса из ЛК

### Шаг 1: Проверить логи браузера

```javascript
// Откройте DevTools → Console
// При выбросе проверьте:
// 1. Есть ли ошибки "401 Unauthorized"
// 2. Есть ли логи "[AuthProvider] Token expired or invalid"
// 3. Есть ли логи "[cmsApi] 401 Unauthorized - clearing auth"
// 4. Есть ли логи "[ecommerceApi] 401 Unauthorized - clearing auth"
```

### Шаг 2: Проверить логи сервера

```bash
# Проверить логи бэкенда на 401 ошибки
pm2 logs primecoder-backend --lines 200 | grep -i "401\|unauthorized\|invalid token"

# Проверить время жизни токена
grep -r "expiresIn" backend/middleware/auth.js
# Должно быть: expiresIn: '7d'
```

### Шаг 3: Проверить токен в localStorage

```javascript
// В консоли браузера:
const token = localStorage.getItem('auth.token');
if (token) {
  // Декодируем JWT токен (без проверки подписи)
  const payload = JSON.parse(atob(token.split('.')[1]));
  const expiresAt = new Date(payload.exp * 1000);
  console.log('Token expires at:', expiresAt);
  console.log('Current time:', new Date());
  console.log('Time until expiry:', expiresAt - new Date(), 'ms');
}
```

### Шаг 4: Проверить useCacheVersionWatcher

```javascript
// Проверить что версия кэша не меняется слишком часто
localStorage.getItem('cache.version')
// Если версия меняется часто - это может вызывать reload
// Но токен должен сохраняться благодаря PRESERVE_KEYS
```

## Решения

### Краткосрочное (уже реализовано):
1. ✅ Глобальная обработка 401
2. ✅ Периодическая проверка валидности токена
3. ✅ Автоматический logout при истечении токена
4. ✅ Исправление цикла редиректов при логине

### Долгосрочное (рекомендуется):
1. **Refresh Token механизм:**
   - Access token: 1 час
   - Refresh token: 30 дней
   - Автоматическое обновление access token

2. **Увеличить время жизни токена:**
   ```javascript
   // В backend/middleware/auth.js
   return jwt.sign(payload, JWT_SECRET, { expiresIn: '30d' }); // Вместо '7d'
   ```
   ⚠️ Не рекомендуется с точки зрения безопасности

3. **Добавить "Remember me" опцию:**
   - Короткая сессия (1 день) без галочки
   - Длинная сессия (30 дней) с галочкой

## Проверка после исправлений

1. **Логин:**
   - ✅ Войти в админ-панель
   - ✅ Не должно быть циклов обновления страницы
   - ✅ Редирект должен быть один раз

2. **Выброс из ЛК:**
   - Проверить логи в консоли браузера при выбросе
   - Проверить логи на сервере
   - Если выброс происходит, проверить причину (401, истечение токена, и т.д.)

3. **Периодическая проверка:**
   - Токен проверяется каждые 5 минут
   - При истечении автоматический logout
