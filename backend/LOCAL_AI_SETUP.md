# 🤖 Настройка локального ИИ

Это руководство поможет настроить использование локальных моделей ИИ вместо OpenAI API.

## Поддерживаемые провайдеры

1. **LM Studio** - простой в использовании, графический интерфейс
2. **Ollama** - командная строка, множество моделей
3. **OpenAI-совместимые API** - любые локальные серверы с OpenAI-совместимым API

## Вариант 1: LM Studio

### Установка и настройка

1. **Скачайте и установите LM Studio:**
   - Сайт: https://lmstudio.ai/
   - Доступно для Windows, macOS, Linux

2. **Загрузите модель:**
   - Откройте LM Studio
   - Перейдите в раздел "Search" и найдите модель (например, "Qwen2.5", "Llama 3", "Mistral")
   - Нажмите "Download" для загрузки модели
   - Рекомендуемые модели:
     - **Qwen2.5-7B-Instruct** - хороший баланс качества и скорости
     - **Llama-3-8B-Instruct** - отличное качество
     - **Mistral-7B-Instruct** - быстрая и эффективная

3. **Запустите локальный сервер:**
   - Перейдите в раздел "Local Server"
   - Выберите загруженную модель
   - Нажмите "Start Server"
   - Сервер запустится на `http://localhost:1234`

4. **Настройте `.env` файл:**

```env
# Использовать локальный AI вместо OpenAI
USE_LOCAL_AI=true

# Провайдер локального AI
LOCAL_AI_PROVIDER=lm-studio

# URL локального AI сервера (по умолчанию для LM Studio)
LOCAL_AI_URL=http://localhost:1234

# Название модели (должно совпадать с названием в LM Studio)
LOCAL_AI_MODEL=Qwen2.5-7B-Instruct

# Параметры генерации
LOCAL_AI_TEMPERATURE=0.7
LOCAL_AI_MAX_TOKENS=2000
```

## Вариант 2: Ollama

### Установка и настройка

1. **Установите Ollama:**
   ```bash
   # macOS / Linux
   curl -fsSL https://ollama.ai/install.sh | sh
   
   # Windows
   # Скачайте установщик с https://ollama.ai/download
   ```

2. **Загрузите модель:**
   ```bash
   # Рекомендуемые модели:
   ollama pull llama3          # Llama 3 8B
   ollama pull qwen2.5:7b      # Qwen 2.5 7B
   ollama pull mistral         # Mistral 7B
   ollama pull phi3            # Phi-3 (легкая модель)
   ```

3. **Запустите Ollama сервер:**
   ```bash
   ollama serve
   ```
   Сервер запустится на `http://localhost:11434`

4. **Настройте `.env` файл:**

```env
# Использовать локальный AI вместо OpenAI
USE_LOCAL_AI=true

# Провайдер локального AI
LOCAL_AI_PROVIDER=ollama

# URL локального AI сервера (по умолчанию для Ollama)
LOCAL_AI_URL=http://localhost:11434

# Название модели (должно совпадать с названием в Ollama)
LOCAL_AI_MODEL=llama3

# Параметры генерации
LOCAL_AI_TEMPERATURE=0.7
```

## Вариант 3: Другие OpenAI-совместимые API

Если у вас есть другой локальный сервер с OpenAI-совместимым API (например, vLLM, text-generation-webui):

```env
USE_LOCAL_AI=true
LOCAL_AI_PROVIDER=openai-compatible
LOCAL_AI_URL=http://localhost:8000  # Ваш локальный сервер
LOCAL_AI_MODEL=your-model-name
LOCAL_AI_TEMPERATURE=0.7
LOCAL_AI_MAX_TOKENS=2000
```

## Проверка работы

После настройки проверьте статус:

```bash
# В админ-панели перейдите в AI Ассистент
# Или сделайте запрос к API:
curl http://localhost:3000/api/ai-chat/status \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Требования к системе

### Минимальные требования:
- **RAM:** 8 GB (для моделей 7B)
- **VRAM:** 4 GB (если используете GPU)
- **Диск:** 5-10 GB свободного места для модели

### Рекомендуемые требования:
- **RAM:** 16 GB+
- **VRAM:** 8 GB+ (для GPU ускорения)
- **CPU:** Современный процессор (Intel i5/i7, AMD Ryzen 5/7)

## Fallback на OpenAI

Если локальный AI недоступен, система автоматически попытается использовать OpenAI (если настроен `OPENAI_API_KEY`). Это полезно для:

- Резервного варианта, если локальный сервер упал
- Сравнения качества ответов
- Тестирования

## Отладка

### Проблема: "LOCAL_AI_CONNECTION_ERROR"

**Решение:**
1. Убедитесь, что локальный сервер запущен
2. Проверьте URL в `.env` файле
3. Проверьте, что порт не занят другим приложением

### Проблема: Медленные ответы

**Решение:**
1. Используйте более легкую модель (например, Phi-3)
2. Уменьшите `LOCAL_AI_MAX_TOKENS`
3. Используйте GPU ускорение (если доступно)

### Проблема: Плохое качество ответов

**Решение:**
1. Используйте более качественную модель (Llama 3, Qwen 2.5)
2. Настройте `LOCAL_AI_TEMPERATURE` (0.7 - баланс, 0.3 - более точные ответы)
3. Увеличьте `LOCAL_AI_MAX_TOKENS` для более развернутых ответов

## Производительность

### Оптимизация для продакшена:

1. **Используйте GPU:**
   - LM Studio автоматически использует GPU, если доступен
   - Ollama: `OLLAMA_NUM_GPU=1 ollama serve`

2. **Настройте кэширование:**
   - LM Studio кэширует контекст автоматически
   - Ollama использует кэш по умолчанию

3. **Ограничьте токены:**
   - Установите разумный `LOCAL_AI_MAX_TOKENS` (1000-2000)

## Безопасность

- Локальный AI работает полностью офлайн
- Данные не отправляются в облако
- Полный контроль над конфиденциальностью
- Можно использовать в закрытых сетях

## Стоимость

- **Локальный AI:** Бесплатно (только электричество)
- **OpenAI API:** ~$0.01-0.03 за 1000 токенов

Для активного использования локальный AI может быть значительно дешевле.
