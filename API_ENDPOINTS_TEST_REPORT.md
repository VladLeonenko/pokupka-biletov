## 📝 ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ

После перезапуска сервера ожидается:

1. **POST /api/consents** - создание согласия:
   - Статус: `200` или `201`
   - Ответ содержит: `id`, `type`, `session_id`, `accepted`, `created_at`

2. **GET /api/consents** - получение согласий:
   - Статус: `200`
   - Ответ: массив согласий

3. **POST /api/consents** (дубликат) - обновление:
   - Статус: `200`
   - Ответ содержит обновленные данные
   - `id` остается тем же (обновление, а не создание)

4. **POST /api/consents** (недопустимый тип) - валидация:
   - Статус: `400`
   - Ответ: `{"error": "Invalid type"}`

5. **POST /api/consents** (без type) - валидация:
   - Статус: `400`
   - Ответ: `{"error": "Type is required"}`
