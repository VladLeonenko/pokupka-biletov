# API и переменные окружения

## Базовый URL API (фронт)

- Утилита: `frontend/src/utils/apiBase.ts`, функция `getApiBase()`.
- Логика: если задан `VITE_API_URL` — используется он (без суффикса `/api`); в dev без него — пустая строка (прокси Vite); в production без него — пустая строка (относительные пути при фронте и бэке на одном домене).
- При сборке для продакшена: если фронт и бэк на одном домене — не задавать `VITE_API_URL`. Если на разных — задать при сборке, например: `VITE_API_URL=https://api.example.com`.

## Бэкенд

- `.env` в `backend/`: `PGUSER`, `PGHOST`, `PGDATABASE`, `PGPASSWORD`, `PGPORT`, плюс при необходимости JWT, CORS origin и т.д. См. существующие `backend/FIX_ENV_*.md`, `FIX_CORS_*.md` до их переноса в wiki.

## CORS

Настройка в `backend/app.js`. Для продакшена при одном домене (фронт как статика того же origin) CORS может не требоваться; при разнесённых доменах — указать разрешённые origin.
