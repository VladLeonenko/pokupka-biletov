# SEO Audit — отчёт по мета-тегам

## Запуск аудита

```bash
npm run seo:audit
```

## Стандарты SEO

| Параметр | Оптимум | Мин | Макс |
|----------|---------|-----|------|
| Title | 50-60 | 30 | 60 |
| Description | 150-160 | 120 | 160 |

## Результаты

### Уникальность

- Все статические страницы имеют уникальные meta title (кроме /politic и /privacy — одна страница, два URL).
- Динамические страницы (товары, блог, кейсы, CMS) берут мета из базы — нужна проверка в админке.

### Страницы с отклонениями от стандартов

**Title слишком длинный (>60 символов):**
- `/` — 87 символов (уменьшить до 60)
- `/catalog` — 62 символа
- `/about` — 64 символа
- `/blog` — 70 символов
- `/ai-team` — 62 символа
- `/reviews` — 69 символов

**Title слишком короткий (<30 символов):**
- Личные кабинеты (noindex) — допустимо
- `/cart`, `/wishlist`, `/search`, `/register` — noindex, допустимо
- `/promotion` — 27 символов

**Description слишком короткая (<120 символов):**
- Многие страницы — расширить до 120-160 символов

### Динамические страницы (проверить в CMS)

| Маршрут | Источник | Что проверить |
|---------|----------|---------------|
| /products/:slug | ProductEditorPage | metaTitle, metaDescription в карточке товара |
| /blog/:slug | BlogEditorPage | seo_title, seo_description в посте |
| /cases/:slug | CaseEditorPage | SEO-поля кейса |
| /:slug | Pages CMS | seo_title, meta_description в редакторе страницы |

### Рекомендации

1. **Индексируемые страницы** — заполнить metaTitle 50-60 символов, metaDescription 120-160.
2. **index.html** — дефолтный title дублирует тему. React перезаписывает при загрузке; для SSR/пререндера нужна синхронизация с HomePage.
3. **Canonical** — SeoMetaTags выставляет через `url`; на /politic и /privacy задать один canonical.
4. **OG-изображения** — для товаров, кейсов, блога используются изображения из CMS (ok).
5. **noindex** — корзина, избранное, личный кабинет, поиск, регистрация — настроено корректно.
