## 🚀 Как использовать

### Шаг 1: Запуск скрипта создания кейсов

```bash
cd backend
node scripts/createCasesForServices.js
```

Скрипт:
- Найдет все активные услуги в базе данных
- Создаст кейсы для услуг, которых еще нет
- Сгенерирует текстовый контент, метрики и инструменты
- Опубликует кейсы (is_published = true)

### Шаг 2: Добавление изображений

После создания кейсов нужно добавить изображения:

1. **Главное изображение (Hero)**:
   - Используйте промпты из `CASE_IMAGES_PROMPTS.md`
   - Загрузите изображения через админ-панель или напрямую в БД
   - Обновите поле `hero_image_url` в таблице `cases`

2. **Галерея изображений**:
   - Добавьте 2 дополнительных изображения в поле `gallery` (JSON массив)
   - Используйте промпты из документации

### Шаг 3: Проверка кейсов

1. Откройте админ-панель → Кейсы
2. Проверьте, что все кейсы созданы и опубликованы
3. Откройте публичную страницу кейса и проверьте:
   - Фулскрин изображение отображается корректно
   - Контейнер выровнен по центру (max-width: 1170px)
   - Контент читаемый и структурированный
   - SEO мета-теги присутствуют

## 📋 Структура кейсов

### Категории кейсов:
- `website` - Разработка сайтов
- `mobile` - Мобильные приложения
- `seo` - SEO продвижение
- `advertising` - Контекстная реклама
- `design` - Дизайн и брендинг
- `ai` - AI решения

### Шаблоны:
- **Полный шаблон** (`templateType: 'full'`) - для сайтов и приложений
  - Использует компоненты: CasesHeaderNew, CasesTasksNew, ColorsSectionNew и др.
  - Фулскрин изображение в первом блоке
  
- **Упрощенный шаблон** (`templateType: 'simple'`) - для SEO, рекламы, дизайна
  - Использует упрощенную структуру с hero-секцией
  - Фулскрин изображение в первом блоке

---

## 🎨 Требования к изображениям

### Hero изображение (главное):
- **Разрешение**: минимум 1920x1080px (16:9)
- **Формат**: PNG или JPG
- **Стиль**: Реалистичные скриншоты или макеты
- **Содержание**: Должно отражать суть проекта

### Галерея (2 изображения):
- **Разрешение**: минимум 1200x800px
- **Формат**: PNG или JPG
- **Содержание**: Дополнительные скриншоты, результаты, интерфейсы

Подробные промпты для каждого типа услуги см. в `CASE_IMAGES_PROMPTS.md`

## 🔧 Технические детали

### Компоненты:
- `CasesHeaderNew` - фулскрин hero-блок с изображением
- `CasePage` - главная страница кейса с роутингом
- `SeoMetaTags` - SEO мета-теги

### Стили:
- Контейнер: `max-width: 1170px`, `margin: 0 auto`, `padding: 0 16px`
- Фулскрин изображение выходит за пределы контейнера
- Адаптивный дизайн для всех устройств

### База данных:
- Таблица `cases` содержит все кейсы
- Поле `category` определяет категорию кейса
- Поле `hero_image_url` - главное изображение
- Поле `gallery` - массив изображений (JSONB)
- Поле `content_json` - дополнительный контент (JSONB)

## ⚠️ Важные замечания

1. **Изображения из портфолио**: Если у вас есть изображения из портфолио (которые вы упомянули), их нужно добавить вручную в соответствующие кейсы через админ-панель или напрямую в БД.

2. **Реальные кейсы**: Для реальных проектов используйте реальные скриншоты и данные.

3. **Выдуманные кейсы**: Для выдуманных кейсов создайте реалистичные макеты и контент, который демонстрирует профессионализм.

4. **SEO**: Все кейсы автоматически получают SEO мета-теги, но при необходимости их можно отредактировать в админ-панели.

## 🐛 Решение проблем

### Кейсы не создаются:
- Проверьте подключение к базе данных
- Убедитесь, что есть активные услуги в таблице `products`
- Проверьте логи скрипта

### Изображения не отображаются:
- Проверьте пути к изображениям
- Убедитесь, что изображения загружены в правильную директорию
- Проверьте права доступа к файлам

### Контейнер не выровнен:
- Проверьте стили в `CasePage.tsx`
- Убедитесь, что нет конфликтующих стилей
- Проверьте в DevTools браузера
