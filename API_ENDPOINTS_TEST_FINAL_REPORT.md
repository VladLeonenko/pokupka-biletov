## 🔒 ENDPOINTS ТРЕБУЮЩИЕ АВТОРИЗАЦИЮ

Следующие endpoints требуют авторизации (токен в заголовке `Authorization: Bearer <token>`):

### PUT /api/consents/:id
- Обновление согласия по ID
- Требует: авторизованный пользователь
- Проверяет: принадлежность согласия пользователю

### DELETE /api/consents/:id
- Отзыв согласия
- Требует: авторизованный пользователь
- Проверяет: принадлежность согласия пользователю

**Примечание:** Эти endpoints не были протестированы в автоматическом режиме, так как требуют авторизации. Для их тестирования нужен:
- Авторизованный пользователь
- JWT токен
- Тестовый скрипт с поддержкой авторизации

## 📝 ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Создание согласия на cookies

```bash
curl -X POST http://localhost:3000/api/consents \
  -H "Content-Type: application/json" \
  -H "x-session-id: my-session-id" \
  -d '{
    "type": "cookies",
    "necessary": true,
    "functional": true,
    "analytical": false,
    "marketing": false,
    "accepted": true
  }'
```

### Получение всех согласий

```bash
curl -X GET http://localhost:3000/api/consents \
  -H "x-session-id: my-session-id"
```

### Обновление согласия

```bash
curl -X POST http://localhost:3000/api/consents \
  -H "Content-Type: application/json" \
  -H "x-session-id: my-session-id" \
  -d '{
    "type": "cookies",
    "analytical": true,
    "marketing": true,
    "accepted": true
  }'
```
