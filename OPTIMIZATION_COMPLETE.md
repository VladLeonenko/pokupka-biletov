## Как это работает

### WebP конвертация
1. Пользователь запрашивает изображение: `/img/photo.jpg`
2. Сервер проверяет заголовок `Accept: image/webp`
3. Если WebP поддерживается:
   - Проверяет существование `photo.webp`
   - Если нет - конвертирует через sharp и сохраняет
   - Отдает WebP версию
4. Если WebP не поддерживается - отдает оригинал

### Service Worker
1. Автоматически регистрируется в production
2. Кэширует статические ресурсы при первой загрузке
3. Использует стратегии:
   - **Cache First** для статики
   - **Network First** для API
   - **Stale-While-Revalidate** для HTML

### Версионирование
- Vite автоматически добавляет хеши к именам файлов
- Версия из `package.json` инжектится в код
- Timestamp сборки для cache busting
