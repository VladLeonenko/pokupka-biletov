Неделя 1: поняли, где теряем заказы

Статистика до проекта:

  • Пн: 18 заказов (9 по телефону, 6 Instagram, 3 личные)
  • Вт: 21 заказ (11 телефон сбросили)
  • Ср: 23 заказа (макс)

Проблемы:

  • Телефон занят → клиент ушёл
  • Меню в PDF → клиент не дочитал
  • Нет доставки → упущенная зона 3км
  • Склад вручную → «роллов нет»

Схема полной автоматизации

Клиент → Сайт/Яндекс.Еда → Корзина → Склад → Кухня → Курьер

В реальном времени: «Роллы есть» ✓ «Курьер в пути» ✓

Стек: React 19 + Vite | Node.js + PostgreSQL | REST API → Яндекс.Еда | Яндекс.Касса + СБП

Неделя 2: запустили MVP сайта

  • Сайт с меню (12 позиций)
  • Корзина + оформление заказа
  • Интеграция склад (есть/нет)
  • Push-уведомления «готов через 18 мин»

Код проверки склада (Node.js):

app.get('/api/menu/:item', async (req, res) => {
  const { stock } = await db.query(
    'SELECT stock FROM inventory WHERE item = $1',
    [req.params.item]
  );
  res.json({ available: stock > 0 });
});
