# Промпты для изображений кейсов

Этот документ содержит промпты и задачи для создания изображений для всех кейсов.

## Структура задач

Для каждого кейса требуется:
1. **Главное изображение (Hero)** - фулскрин изображение для первого блока
2. **2 дополнительных изображения** - для галереи проекта

---

## Кейсы по категориям

### Разработка сайтов (website)

#### Кейс: Разработка сайта под ключ
- **Hero изображение**: Скриншот современного корпоративного сайта на ноутбуке и планшете, светлый дизайн, профессиональный интерфейс, элементы навигации и контент видны
- **Изображение 1**: Скриншот мобильной версии сайта на смартфоне, адаптивный дизайн
- **Изображение 2**: Скриншот админ-панели или системы управления контентом

#### Кейс: Создание интернет-магазина
- **Hero изображение**: Скриншот интернет-магазина с каталогом товаров, корзиной покупок, на ноутбуке и мобильном устройстве
- **Изображение 1**: Скриншот страницы товара с фотографиями, описанием, ценами и кнопкой "Добавить в корзину"
- **Изображение 2**: Скриншот страницы оформления заказа с формой доставки и оплаты

#### Кейс: Разработка корпоративного сайта
- **Hero изображение**: Скриншот корпоративного сайта компании, современный дизайн, логотип, меню навигации, на десктопе и планшете
- **Изображение 1**: Скриншот раздела "О компании" или "Услуги" с контентом
- **Изображение 2**: Скриншот контактной формы или раздела "Контакты"

#### Кейс: Разработка лендингов
- **Hero изображение**: Скриншот лендинг-страницы с ярким заголовком, призывом к действию, форма захвата контактов, на ноутбуке
- **Изображение 1**: Скриншот блока с преимуществами или отзывами
- **Изображение 2**: Скриншот мобильной версии лендинга

#### Кейс: Разработка интернет-магазина (E-commerce)
- **Hero изображение**: Скриншот интернет-магазина с категориями товаров, фильтрами, поиском, на десктопе и мобильном
- **Изображение 1**: Скриншот страницы категории с товарами в сетке
- **Изображение 2**: Скриншот личного кабинета пользователя с заказами

#### Кейс: Разработка сайта ресторана/кафе
- **Hero изображение**: Скриншот сайта ресторана с меню, фотографиями блюд, формой бронирования столика, на планшете
- **Изображение 1**: Скриншот онлайн-меню с блюдами и ценами
- **Изображение 2**: Скриншот формы заказа доставки еды

#### Кейс: Разработка образовательных платформ
- **Hero изображение**: Скриншот образовательной платформы с курсами, видео-лекциями, на ноутбуке и планшете
- **Изображение 1**: Скриншот страницы курса с уроками и прогрессом
- **Изображение 2**: Скриншот личного кабинета студента с достижениями

#### Кейс: Разработка личного кабинета
- **Hero изображение**: Скриншот личного кабинета пользователя с дашбордом, статистикой, на десктопе
- **Изображение 1**: Скриншот раздела настроек профиля
- **Изображение 2**: Скриншот мобильной версии личного кабинета

### Мобильные приложения (mobile)

#### Кейс: Разработка приложений
- **Hero изображение**: Скриншоты мобильного приложения на iPhone и Android устройстве, современный UI/UX дизайн

- **Изображение 2**: Скриншот отчета по SEO-оптимизации с метриками

#### Кейс: SEO-продвижение

- **Изображение 2**: Скриншот структурированных данных или schema.org разметки

## Общие требования к изображениям

### Технические требования:
- **Формат**: PNG или JPG
- **Разрешение**: 
  - Hero изображение: минимум 1920x1080px (16:9)
  - Дополнительные изображения: минимум 1200x800px
- **Качество**: Высокое, без сжатия артефактов
- **Стиль**: Современный, профессиональный, соответствует бренду PrimeCoder

### Содержание:
- Изображения должны быть реалистичными скриншотами или макетами
- Показывать реальные интерфейсы, инструменты, результаты работы
- Демонстрировать профессионализм и качество услуг
- Быть релевантными к конкретному кейсу

### Использование:
- Hero изображения используются как фулскрин в первом блоке страницы кейса
- Дополнительные изображения добавляются в галерею проекта
- Все изображения должны иметь альтернативный текст для SEO

---

## Примечания

1. Для каждого кейса из базы данных нужно создать соответствующие изображения
2. Если кейс относится к реальному проекту, используйте реальные скриншоты
3. Если кейс выдуманный, создайте реалистичные макеты и скриншоты
4. Все изображения должны быть оптимизированы для веб-использования
5. Рекомендуется использовать единый стиль для всех изображений одного типа
