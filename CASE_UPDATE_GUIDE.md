# Руководство по комплексному обновлению кейсов

## Что делает скрипт

Скрипт `comprehensiveCaseUpdate.js` выполняет комплексное обновление всех кейсов:

1. **Обновляет доноров** - заменяет топовые сайты на более скромные (2-5 страница выдачи)
2. **Добавляет название компании** - извлекает из URL и добавляет в название кейса
3. **Заполняет текстовые блоки** - автоматически генерирует Задачи, Решения, Результаты
4. **Извлекает цвета** - автоматически определяет цветовую палитру сайта
5. **Извлекает типографику** - определяет используемые шрифты
6. **Генерирует метрики** - создает индивидуальные метрики для каждого кейса
7. **Создает скриншоты** - делает обложки с обходом блокировок

## Запуск скрипта

### Тестовый запуск (3 кейса)
```bash
cd backend
node scripts/comprehensiveCaseUpdate.js --test
```

### Полный запуск (все кейсы)
```bash
cd backend
node scripts/comprehensiveCaseUpdate.js
```

## Что обновляется

### 1. Доноры
- Заменяются на более скромные сайты из регионов
- Для каждой категории свой список доноров
- Не используются топовые сайты (Яндекс, Google, Avito и т.д.)

### 2. Названия кейсов
- Добавляется название компании: `"Название кейса для Название Компании"`
- Название извлекается из URL донора

### 3. Текстовые блоки
- **Задача** - генерируется на основе категории и названия компании
- **Решение** - описывает что было сделано
- **Результаты** - описывает достигнутые результаты

### 4. Цвета
- Автоматически извлекаются из сайта донора
- Отображаются в виде кружков с hex кодами
- Сохраняются в `contentJson.colors.palette`

### 5. Типографика
- Автоматически определяется шрифт сайта
- Отображается в разных размерах с lorem текстом
- Сохраняется в `contentJson.typography`

### 6. Метрики
- **Дни** - случайное значение для каждого кейса (15-150 дней в зависимости от категории)
- **Второй показатель** - зависит от категории:
  - Website: количество страниц
  - Mobile: количество функций
  - SEO: количество запросов в топ-10
  - Advertising: количество кампаний
  - Design: количество элементов
  - AI: количество автоматизаций

### 7. Скриншоты
- Создаются с обходом блокировок (User-Agent, задержки)
- Сохраняются в `frontend/public/legacy/img/cases/{slug}/cover.png`
- Проверяется на блокировки (Cloudflare, captcha)

## Обход блокировок

Скрипт использует несколько методов для обхода блокировок:

1. **User-Agent** - устанавливается обычный браузерный User-Agent
2. **HTTP Headers** - добавляются стандартные заголовки браузера
3. **Задержки** - пауза 5 секунд между запросами
4. **Проверка блокировок** - проверяется наличие Cloudflare, captcha и т.д.

## Время выполнения

- **Тестовый запуск (3 кейса)**: ~2-3 минуты
- **Полный запуск (80+ кейсов)**: ~2-3 часа

Время зависит от:
- Скорости загрузки сайтов доноров
- Наличия блокировок
- Количества кейсов

## Ошибки

Если скрипт встречает ошибки:

1. **Блокировка сайта** - скрипт пропускает кейс и продолжает работу
2. **Ошибка загрузки** - пробуются разные стратегии загрузки
3. **Отсутствие файлов** - создаются директории автоматически

Все ошибки логируются в консоль, но не останавливают выполнение скрипта.

## После запуска

После успешного выполнения скрипта:

1. ✅ Все кейсы обновлены с новыми донорами
2. ✅ Названия содержат названия компаний
3. ✅ Текстовые блоки заполнены
4. ✅ Цвета и типографика извлечены
5. ✅ Метрики индивидуализированы
6. ✅ Скриншоты созданы

## Проверка результатов

1. Откройте любой кейс в админке
2. Проверьте название - должно содержать название компании
3. Проверьте донора - должен быть региональный сайт
4. Проверьте текстовые блоки - должны быть заполнены
5. Проверьте цвета - должны отображаться кружки с hex
6. Проверьте типографику - должен быть шрифт в разных размерах
7. Проверьте метрики - должны быть индивидуальные значения
