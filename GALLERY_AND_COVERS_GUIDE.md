# Руководство по созданию галереи и обложек для кейсов

## Структура файлов

Все изображения для кейсов должны храниться в следующей структуре:

```
frontend/public/legacy/img/cases/
  ├── {slug-case-1}/
  │   ├── cover.png          # Обложка кейса
  │   ├── gallery-1.png      # Скриншот 1 (главная страница)
  │   ├── gallery-2.png        # Скриншот 2 (каталог/услуги)
  │   └── gallery-3.png        # Скриншот 3 (контакты/форма)
  ├── {slug-case-2}/
  │   ├── cover.png
  │   ├── gallery-1.png
  │   ├── gallery-2.png
  │   └── gallery-3.png
  └── ...
```

## Требования к изображениям

### Обложка (cover.png)
- **Размер**: 1920x1080px (16:9)
- **Формат**: PNG или JPG
- **Содержание**: Главный скриншот сайта донора (главная страница)
- **Качество**: Высокое разрешение, четкое изображение

### Галерея (gallery-1.png, gallery-2.png, gallery-3.png)
- **Размер**: 1920x1080px (16:9) или пропорционально
- **Формат**: PNG или JPG
- **Содержание**:
  - **gallery-1.png**: Главная страница сайта донора
  - **gallery-2.png**: Каталог товаров/услуг или внутренняя страница
  - **gallery-3.png**: Контакты, форма обратной связи или другая важная страница
- **Качество**: Высокое разрешение, четкое изображение

## Как создать скриншоты

### Вариант 1: Ручное создание
1. Откройте сайт донора в браузере
2. Сделайте скриншот нужной страницы
3. Обрежьте и обработайте изображение до нужного размера
4. Сохраните в соответствующую папку

### Вариант 2: Использование сервисов скриншотов
Можно использовать API сервисов для автоматического создания скриншотов:
- **ScreenshotAPI**: https://screenshotapi.net/
- **URLBox**: https://urlbox.io/
- **htmlcsstoimage.com**: https://htmlcsstoimage.com/

### Вариант 3: Скрипт для автоматизации
Можно создать скрипт, который будет автоматически делать скриншоты сайтов доноров.

## Пример скрипта для создания скриншотов

```javascript
// Пример использования Puppeteer для создания скриншотов
const puppeteer = require('puppeteer');

async function createScreenshots(donorUrl, slug) {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  
  // Обложка - главная страница
  await page.goto(donorUrl);
  await page.screenshot({ 
    path: `frontend/public/legacy/img/cases/${slug}/cover.png`,
    fullPage: true 
  });
  
  // Галерея 1 - главная страница
  await page.screenshot({ 
    path: `frontend/public/legacy/img/cases/${slug}/gallery-1.png`,
    fullPage: true 
  });
  
  // Галерея 2 - каталог/услуги (нужно найти URL)
  // await page.goto(donorUrl + '/catalog');
  // await page.screenshot({ 
  //   path: `frontend/public/legacy/img/cases/${slug}/gallery-2.png`,
  //   fullPage: true 
  // });
  
  await browser.close();
}
```

## Проверка доноров

Перед созданием скриншотов убедитесь, что:
1. Донор релевантен категории кейса:
   - **Реклама**: Компании/фирмы/ИП/селлеры, которым мы помогли с рекламой
   - **Дизайн**: Компании, которым мы помогли с дизайном
   - **SEO**: Компании, которым мы помогли с SEO
   - **Сайты**: Менее известные сайты из разных популярных ниш (не банки)
2. Сайт донора доступен и работает
3. Сайт имеет релевантный контент для кейса

## Важные замечания

- **Не используйте популярные сайты** (ВК, Авито, Яндекс Маркет) как доноры
- **Для рекламы/дизайна/SEO** доноры должны быть компаниями, которым мы предоставляли услуги
- **Для сайтов** используйте менее известные, но популярные сайты из разных ниш
- **Банки не подходят** для кейсов разработки сайтов (у них свои IT отделы)

## Следующие шаги

1. Запустите скрипт `generateGalleryAndCovers.js` для генерации URL
2. Создайте папки для каждого кейса: `frontend/public/legacy/img/cases/{slug}/`
3. Создайте скриншоты для каждого кейса
4. Сохраните изображения в соответствующие папки
5. Проверьте, что все изображения загружаются корректно
