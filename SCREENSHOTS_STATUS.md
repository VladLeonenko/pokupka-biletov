## 🔍 Детали работы скрипта

### Скрипт
- **Файл:** `backend/scripts/createScreenshots.js`
- **Технология:** Playwright (Chromium)
- **Режим:** Headless (без GUI)

### Процесс работы

1. Скрипт получает все опубликованные кейсы с `donor_url` из базы данных
2. Для каждого кейса создает директорию `frontend/public/legacy/img/cases/{slug}/`
3. Создает 4 изображения:
   - `cover.png` - обложка (главная страница донора)
   - `gallery-1.png` - первое изображение галереи (главная страница)
   - `gallery-2.png` - второе изображение (каталог/услуги)
   - `gallery-3.png` - третье изображение (контакты/о нас)

### Стратегия загрузки

Скрипт использует несколько стратегий для загрузки страниц:
1. `networkidle` (30 сек) - ожидание завершения сетевых запросов
2. `domcontentloaded` (15 сек) - ожидание загрузки DOM
3. `load` (10 сек) - ожидание загрузки страницы

Для gallery-2 и gallery-3 скрипт пытается найти подходящие страницы:
- `/catalog`, `/catalogue`, `/products`, `/services`, `/uslugi`, `/katalog`, `/shop`
- `/contacts`, `/contact`, `/kontakty`, `/about`, `/o-nas`

### Обработка ошибок

- Если страница не загружается, скрипт пробует альтернативные URL
- Если все попытки неудачны, скрипт пропускает изображение и продолжает работу
- Ошибки не останавливают выполнение скрипта
